Smart Monitor

IoT Baby Monitoring System with Real-time Audio/Video Analysis

System Overview

Smart Monitor is an advanced IoT baby monitoring system that provides real-time audio and video analysis with physics-based simulation. The system correlates sensor data to detect baby states and provide intelligent alerts.

Key Features

  • Real-time audio analysis with crying detection
  • Video motion detection with probability scoring
  • Physics-based baby state simulation
  • Correlated sensor data (temperature, humidity, light)
  • Web-based dashboard with live updates
  • RESTful API for integration
  • Nightlight automation

Technology Stack

C/C++ (Core), Rust (Motion Detection), JavaScript (Frontend), WebSocket (Real-time), HTTP/REST API, JSON Protocol, Linux/Embedded, GStreamer (Audio), OpenCV (Video), Physics Simulation
100%

System Architecture

The Smart Monitor consists of several interconnected components that work together to provide comprehensive baby monitoring capabilities.

Core Components

  • Data Agent - Central simulation and data processing engine
  • HTTP Server - RESTful API and web interface
  • Protocol Server - Real-time WebSocket communication
  • Motion Detector - Rust-based video analysis
  • Audio Processor - Noise detection and crying analysis
  • Sensor Manager - Temperature, humidity, light sensors
100%

API Documentation

The Smart Monitor provides RESTful HTTP endpoints and real-time WebSocket connections for accessing system data and controlling functionality.

HTTP Endpoints

GET /sensors/i2c - Sensor data (temp, humidity, light, motion) GET /audio/status - Audio analysis (noise, voice, crying, screaming) GET /video/status - Video analysis (motion probability, detection) GET /system/status - System metrics (CPU, memory, uptime) POST /api/command - Send commands to system

Data Flow

  • Client requests data via HTTP endpoints every 100ms
  • Server returns JSON with current sensor readings
  • WebSocket provides real-time updates for critical events
  • All data is correlated through baby state machine
100%

Data Flow & Processing

Understanding how data flows through the system is crucial for integration and debugging.

Processing Pipeline

  • Sensor data collection at 10Hz frequency
  • Baby state machine with realistic transitions
  • Physics-based correlation between sensors
  • RC low-pass filtering for smooth data
  • Probabilistic detection thresholds

State Transitions

SLEEPING_PEACEFUL → RESTLESS → STIRRING → FUSSY → CRYING_SOFT → CRYING_LOUD → SCREAMING → FALLING_ASLEEP → SLEEPING_PEACEFUL
100%

Deployment & Operations

Smart Monitor is designed for embedded Linux systems with comprehensive deployment and monitoring capabilities.

Build System

  • CMake build system with cross-compilation support
  • Ninja build tool for fast compilation
  • Rust integration for motion detection
  • Dependency management through pkg-config

Runtime Requirements

Linux kernel 4.15+, 2GB RAM minimum, 10GB storage space, Network connectivity, Camera device (/dev/video0), Audio input device
